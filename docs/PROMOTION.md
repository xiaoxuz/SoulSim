# SoulSim 项目推广文档

## 1. 项目定位

SoulSim 是一个面向复杂议题推演的多 Agent 社会模拟平台。它不是普通聊天机器人，也不是单轮剧本生成器，而是一个可以从“世界设定”出发，自动生成角色、关系、事件、记忆、报告和互动讨论的模拟工作台。

用户只需要输入一个现实或虚构议题，例如：

- AI 教育产品进入重点中学后，各方如何博弈？
- 末日废土中，人类会合作重建还是走向毁灭？
- 创业团队在关键 pivot 节点上如何分裂或达成共识？
- 社区谣言如何扩散、变形和消退？

SoulSim 会自动构建一个可推演的社会场景，让多个具有不同立场、目标和性格的 Agent 在同一个世界里连续行动、表达、改变关系，并最终形成结构化报告。

## 2. 核心价值

### 2.1 把抽象议题变成可观察的社会实验

很多复杂问题难在“变量太多、角色太多、冲突太隐蔽”。SoulSim 将问题拆成：

- 世界背景
- 关键角色
- 利益关系
- 事件演化
- 记忆与态度变化
- 群体态势
- 指标变化

用户可以像观察实验一样，看不同角色在连续事件中如何反应。

### 2.2 角色不是模板，而是带灵魂的 Agent

每个 Agent 都有自己的：

- 身份和背景
- 初始立场和目标
- 心智模型
- 决策规则
- 表达风格
- 价值观
- 反模式
- 诚实边界

系统支持粘贴 `SKILL.md` 形式的灵魂配置，为角色注入更强的人格一致性。角色在推演、1v1 对话和群聊中都会尽量保持自己的语言指纹与判断方式。

```

### 2.3 一次世界，多次推演

同一个世界可以进行多次模拟。每次模拟都是一条独立的推演记录：

- 可以从同一个世界 baseline 重新开始。
- 可以比较不同 run 的事件走向。
- 报告和互动绑定具体 run。
- 旧 run 的报告、聊天、日志不会串到新 run。

这让 SoulSim 不只是一次性生成结果，而是可以做“对照实验”。

```

### 2.4 推演过程透明可追踪

SoulSim 不只给最终结论，还展示推演过程：

- 每天发生了什么事件。
- 每个 Agent 如何理解事件。
- 每个 Agent 的内心活动、对外发言和实际行动。
- 世界状态如何变化。
- 群体态势、指标和关系如何演化。
- 上帝干预在什么时间点进入，造成什么影响。

这让结果更容易复盘，也更适合做研究、产品策略、剧本设计、组织决策预演。

### 2.5 报告不是总结，而是结构化洞察

推演结束后，SoulSim 可以生成完整报告，包括：

- 故事化叙事
- 执行摘要
- 目标达成评估
- 世界设定复盘
- 事件时间线
- 角色视角
- 关系变化
- 指标变化
- 关键驱动因素
- 证据映射
- 干预影响分析

用户可以快速从“发生了什么”进入“为什么会这样”。

### 2.6 推演之后还能继续追问

SoulSim 的深度互动不是另起一个普通聊天，而是基于某次推演记录继续对话：

- 和某个角色 1v1 深聊。
- 和 Report Agent 追问报告。
- 创建多 Agent 群聊，观察角色互相回应、质疑和沉默。

这让用户可以从宏观报告深入到具体角色的主观视角。

## 3. 典型使用场景

### 3.1 产品策略预演

适合问题：

- 一个新产品进入学校、企业、社区后，不同利益方如何反应？
- 产品推广会遇到哪些隐藏阻力？
- 哪些角色会成为支持者，哪些角色会成为阻力源？
- 哪些指标改善会推动扩张，哪些风险会触发叫停？

示例：AI 教育产品进入重点中学。系统可以模拟校长、语文老师、数学老师、家长代表、销售经理之间的协商与冲突。

### 3.2 组织决策推演

适合问题：

- 创业团队是否 pivot？
- 管理层推动新制度时，一线如何反应？
- 组织危机中，不同角色如何站队？
- 决策信息逐步释放后，共识如何形成或崩塌？

SoulSim 可以帮助用户提前观察组织内部的立场演化。

### 3.3 舆情和谣言传播模拟

适合问题：

- 一条消息如何在社区中传播？
- 谁会相信，谁会质疑，谁会放大？
- 哪些干预能降低误解？
- 信任如何影响信息扩散？

系统可用于传播机制分析和干预策略设计。

### 3.4 剧本、游戏和世界观创作

适合问题：

- 一个设定下，角色关系如何自然演化？
- 冲突如何从小事件升级？
- 世界状态如何随角色行动变化？
- 如何生成更可信的群像故事？

SoulSim 可以作为故事引擎，帮助创作者生成有因果链的剧情素材。

### 3.5 教育和研究

适合问题：

- 复杂系统如何表现出涌现行为？
- 个体目标如何影响群体结果？
- 干预时机如何改变系统走向？
- 多 Agent 模拟如何用于课堂讨论？

系统可以作为社会科学、组织行为、AI Agent 研究的实验台。

## 5. 产品亮点

### 5.1 多 Agent 不是并排聊天，而是共享时空

所有 Agent 在同一个事件和世界状态下行动。后发言的 Agent 能看到前面角色已经发生的行动，从而避免互相冲突。

### 5.2 推演记录独立，适合对照实验

同一世界可以多次推演，每次 run 都有自己的日志、报告、聊天和干预记录。用户可以比较不同路径下的系统演化。

### 5.3 支持上帝干预

用户可以在推演过程中释放信息、修改条件或施加干预，观察系统如何变化。

### 5.4 报告和聊天绑定具体 run

报告、1v1、群聊都基于当前选中的推演记录，不会把不同 run 的内容混在一起。

### 5.5 可解释的过程数据

每一步都有日志、事件、角色回应、世界状态、记忆和图谱变化，方便复盘和调试。

## 6. 和普通 AI 对话工具的区别

| 对比项 | 普通聊天机器人 | SoulSim |
| --- | --- | --- |
| 输入 | 单轮问题 | 世界种子 + 推演目标 |
| 输出 | 一段回答 | 世界、角色、事件、日志、报告、互动 |
| 角色 | 临时扮演 | 持久 Agent + SkillProfile |
| 时间 | 单次回答 | 多日连续推演 |
| 关系 | 主要靠文本描述 | 知识图谱 + 记忆 + 状态 |
| 复盘 | 难追踪 | 时间线、日志、报告、证据 |
| 对照实验 | 不方便 | 同一 world 多 run |
| 后续追问 | 重新上下文 | 绑定具体 run 深度互动 |

## 9. 对外一句话介绍

SoulSim 是一个多 Agent 社会模拟工作台：输入一个复杂议题，它会生成一个世界，让不同立场的角色连续互动、冲突、协商和演化，最终产出可追踪的过程日志、结构化报告和可继续追问的深度互动。

## 10. 演示建议话术

你可以这样介绍：

> 传统 AI 工具擅长回答问题，但复杂社会问题不是一句话能回答的。SoulSim 把问题变成一个可运行的世界：它生成角色、关系和事件，让不同立场的 Agent 在同一时空里连续行动。我们不仅能看到结论，还能看到结论是怎么一步步形成的。

然后按这个顺序演示：

1. 展示首页和项目定位。
2. 输入一个真实议题，例如“AI 教育产品进入重点中学”。
3. 展示自动生成的 Agent 和知识图谱。
4. 运行模拟，看角色如何分歧。
5. 展示报告，强调结构化洞察。
6. 进入群聊，向角色追问关键问题。
7. 回到 Step 2，展示同一 world 可多次推演和不同 run 的隔离。
